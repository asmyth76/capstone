About the Project

This text prediction application is the final project of the Johns Hopkins Data Science Specialization for the Capstone.

The corpora used to build the prediction was collected from publicly available sources - news, blog posts and twitter posts. This data was provided by SwiftKey.

I started off using the package tm, then ngram and eventually landed on quanteda. I also used the following packages: data.table, stringr, qdapRegx and ggplot2.

The Data

I began by downloading the data and after some exploration, cleaned it up a bit. I removed profanity, stop words, punctuation, numbers and symbols. I should note that I only used 5% of the data to create the predictions.

Next came tokens and then splitting them up into ngrams. I opted for unigrams, bigrams and trigrams (mainly due to my laptop's age and processing ability). Following the creation of the ngrams I stemmed the data.

The Algorithm

Next I moved the ngrams into a document feature matrix and pruned the DFMs where the minimum term frequency was set to 3. Pruning the dataset no doubt lowered the accuracy of the predictive application, but it was a tradeoff for the speed needed in this application. In addition, I also used data tables versus dataframes and indexed them as well. This has proved to have the desired impact on the application as the predictions come back instaneously.

Kneser Ney smoothing was implemented to calculate the probability of the ngrams.

Predictive Application

The app is fairly simple. Depending on what is entered, first it will look in the trigrams and if nothing is found, move to bigrams and lastly take a sample from the unigrams data table. The top 3 words are returned to the user.
